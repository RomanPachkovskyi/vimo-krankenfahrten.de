## Що дозволено

- Редагувати контент сайту через `site/src/config.ts` — єдине місце для текстів, SEO, контактів, послуг, FAQ.
- Редагувати компоненти у `site/src/` (React/TypeScript).
- Запускати dev-сервер: `cd site && npm run dev` (порт 8080) або `docker compose up` (порт 8083).
- Запускати тести: `cd site && npm test`.
- Виконувати build: `cd site && npm run build`.
- Читати та оновлювати [CHANGELOG.md](CHANGELOG.md) після кожної значної зміни.
- Комітити зміни у гілку `main` — CI автоматично деплоїть на GitHub Pages.

## Що заборонено

- **Не пушити force** (`git push --force`) у `main` без явного запиту власника.
- **Не видаляти** `site/dist/` вручну — артефакт збірки, потрібен для FTP-деплою.
- **Не комітити** `.env` — файл з FTP-кредешнлами, він у `.gitignore`.
- **Не змінювати** GitHub Actions workflow (`.github/workflows/deploy.yml`) без обговорення.
- **Не встановлювати** нові npm-залежності без явної потреби.
- **Не розбивати** сайт на кілька сторінок — це навмисно one-page.
- **Не додавати** зовнішніх трекерів, аналітики, рекламних скриптів — порушення CSP і Datenschutz.
- Не редагувати `httpdocs/` вручну — ця тека синхронізується деплоєм, не є джерелом істини.

## Порядок роботи

1. Уточни завдання в [PROJECT.md](PROJECT.md) (стек, структура, обмеження).
2. Зміни контент — лише через `site/src/config.ts`.
3. Перевір: `npm test` + `npm run build` мають пройти без помилок.
4. Після значних змін додай запис до [CHANGELOG.md](CHANGELOG.md).
5. Коміт у `main` — деплой відбудеться автоматично.
